LZ77 Sliding Window Codec — Implementation Specification
Version: 1.0
Component: Data Archival Pipeline, Compression Layer

Parameters
----------
WINDOW_SIZE    = 255  (bytes) — size of the lookback search buffer
LOOKAHEAD_SIZE = 15   (bytes) — maximum length of a matched sequence

Token Format
------------
Each output token is exactly 3 bytes:
  Byte 0: offset  — distance back into the window where the match starts (0=no match)
  Byte 1: length  — number of bytes matched from the window (0=no match)
  Byte 2: next_char — the next literal byte after the match (always present)

When offset=0 and length=0, the token is a literal: next_char is the raw byte.
When offset>0, the token references a back-copy followed by one new byte.

Overlapping Back-References
----------------------------
Overlapping matches MUST be supported. If offset=1 and length=4, the decoder
copies 4 bytes starting 1 byte back, which repeats the last byte 4 times.
The implementation must correctly handle the case where the match length
exceeds the offset distance.

Round-Trip Correctness
-----------------------
The codec MUST satisfy: decode(encode(data)) == data for all input strings.
This is the primary correctness invariant and is verified by the test suite.

Compression Effectiveness
--------------------------
For inputs with repeated substrings longer than 3 bytes, the encoded byte
count MUST be less than len(data.encode()) (i.e., actual compression occurs).
